# Лабораторная работа 5

## **Docker: Docker-compose и мультирепозиторий**

Используемое приложение: https://gitlab.com/deusops/projects/react-spring-app

Цель лабораторной: освоить запуск полноценного веб-приложения из разных репозиториев. Приложение должно содержать фронтенд и бэкэнд части из соответствующих репозиториев. Компоуз-файл должен запускать базу данных для приложения, и обратный прокси nginx

## Требования

1. Использовать базовые легковесный образ – alpine
2. Учесть запуск `backend`, `frontend`, `db` и `nginx-proxy`
3. Вся конфигурация выполняется через переменные окружения, передающиеся через `env_file` и `environment`
4. Точкой входа является только `nginx-proxy`, остальные сервисы не имеют открытых наружу портов
5. Должна быть возможность конфигурирования через файлы настроек в виде `volume`
6. Контейнеры должны запускаться от непривилегированного пользователя
7. После установки всех нужных утилит, должен очищаться кеш


### Запуск приложения

1. Скопировать `.env.example` в `.env` и настроить переменные при необходимости:
```bash
cp .env.example .env
```

2. Запустить приложение:
```bash
docker-compose up --build
```

3. Приложение будет доступно по адресу: `http://localhost:3000`

### Endpoints
- Frontend: `http://localhost:3000/`
- Backend API: `http://localhost:3000/api/v1/`

### Остановка приложения
```bash
docker-compose down
```

### Удаление с очисткой volumes
```bash
docker-compose down -v
```
